Phase 1 — Prerequisites
Install tools on your computer (one-time setup)
Steps 1–3
1
Install Git on your computer
Version control tool — needed to push code to GitHub
Git ▶

Git lets you save and upload your code to GitHub. Check if you already have it first:

git --version

If it shows a version number (e.g. git version 2.x.x) you're done — skip to step 2!

If not, download and install from:

🔗Windows: https://git-scm.com/download/win — run the installer, click Next on everything
Mac: Run xcode-select --install in Terminal
Linux: sudo apt install git
2
Install Terraform
The tool that creates AWS resources automatically
Terraform ▶

Terraform reads your .tf files and builds AWS infrastructure for you. Download it from:

🔗All platforms: https://developer.hashicorp.com/terraform/downloads
Download, unzip, and move it to a folder in your PATH (the installer does this automatically on Windows).

Verify installation:

terraform --version
⚠️You need Terraform v1.3.0 or higher. If older, re-download.
3
Install AWS CLI
Command-line tool to talk to your AWS account
AWS ▶

The AWS CLI lets Terraform and GitHub Actions communicate with your AWS sandbox account.

🔗Download: https://aws.amazon.com/cli/
Windows: Run the .msi installer | Mac: Run the .pkg installer
aws --version

You should see something like aws-cli/2.x.x.

☁️
Phase 2 — AWS Setup
Get your AWS sandbox credentials
Steps 4–5
4
Get your AWS Access Keys
Credentials so tools can talk to your AWS sandbox
AWS IAM ▶

Follow these steps inside the AWS console:

#What to do
1Log in to your AWS sandbox console at console.aws.amazon.com
2Click your username (top right) → Security credentials
3Scroll to "Access keys" section → click "Create access key"
4Select "CLI" as the use case → click Next → Create
5Copy both values and save them somewhere safe (notepad is fine)
ValueLooks like
AWS_ACCESS_KEY_IDAKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEYwJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
⚠️You will never see the secret key again after closing this page. Copy it now!
5
Configure AWS CLI locally
So Terraform can run from your machine too (optional but useful)
AWS CLI ▶

Open your terminal and run:

aws configure

It will ask 4 questions. Answer them like this:

AWS Access Key ID → paste your Access Key ID AWS Secret Access Key → paste your Secret Key Default region name → us-east-1 Default output format → json

Test it works:

aws s3 ls

No error = you're connected to AWS! ✅

🐙
Phase 3 — GitHub Setup
Create your repo and add secrets
Steps 6–7
6
Create a new GitHub repository
This is where all your POC files will live
GitHub ▶
#What to do
1Go to github.com and log in with your personal account
2Click the "+" button (top right) → "New repository"
3Name it: aws-static-poc
4Set visibility: Private (safer for sandbox credentials)
5Check "Add a README file" → Click "Create repository"
💡You'll now have an empty repo at github.com/YOUR_USERNAME/aws-static-poc
7
Add AWS secrets to GitHub
So GitHub Actions can access your AWS account securely
Secrets ▶

GitHub Secrets store your credentials safely — the pipeline reads them without exposing them.

#What to do
1Inside your repo → click Settings tab
2Left sidebar → Secrets and variables → Actions
3Click "New repository secret"
4Add the first secret (name + value from Step 4)
5Click "Add secret" → repeat for the second
Secret Name (exact!)Value
AWS_ACCESS_KEY_IDYour Access Key ID from Step 4
AWS_SECRET_ACCESS_KEYYour Secret Key from Step 4
⚠️Secret names are case-sensitive. Type them exactly as shown above.
📁
Phase 4 — Upload POC Files
Clone the repo and add your project files
Steps 8–9
8
Clone the repo to your computer
Download your GitHub repo as a local folder
Git ▶

Open a terminal (Command Prompt / PowerShell / Terminal) and run:

# Replace YOUR_USERNAME with your GitHub username
git clone https://github.com/YOUR_USERNAME/aws-static-poc.git
cd aws-static-poc

You now have a local folder called aws-static-poc on your computer.

9
Copy POC files into the folder
Add all the generated files into the correct folder structure
Files ▶

Create this exact folder structure inside aws-static-poc/:

aws-static-poc/ ├── index.html ← your website file ├── terraform/ │ ├── main.tf ← AWS infrastructure │ ├── variables.tf ← configuration │ └── outputs.tf ← prints your URL └── .github/ └── workflows/ └── deploy.yml ← pipeline
💡All 6 files were generated for you in the previous step. Just copy them from the download into this folder structure.
⚠️The .github folder starts with a dot — make sure it's named exactly .github (not github)
🚀
Phase 5 — Deploy!
Push code and watch it go live automatically
Steps 10–12
10
Push code to GitHub
This triggers the pipeline automatically
Git Push ▶

In your terminal, inside the aws-static-poc folder, run these 3 commands:

# Stage all files
git add .

# Save a snapshot
git commit -m "Initial POC deployment"

# Upload to GitHub → triggers pipeline!
git push origin main
✅The moment you push, GitHub Actions starts running automatically. You don't need to do anything else!
11
Watch the pipeline run
Monitor each step in GitHub Actions (takes ~5 minutes)
Monitoring ▶
#What to do
1Go to your repo on github.com
2Click the Actions tab (top navigation)
3You'll see a workflow called "Deploy to AWS" running (orange dot = in progress)
4Click it to see each step in real-time

You'll see these steps completing one by one:

✓ Checkout repository ✓ Configure AWS credentials ✓ Setup Terraform ✓ Terraform Init ✓ Terraform Plan ✓ Terraform Apply ← creates S3 + CloudFront ✓ Get Terraform outputs ✓ Upload website to S3 ✓ Invalidate CloudFront cache ✓ Print website URL ← your URL is here!
⏱️This takes about 3–7 minutes on the first run. CloudFront setup takes the longest.
12
Open your live website! 🎉
Find your URL and see your site live on the internet
🌐 LIVE ▶

Once all pipeline steps show a ✅ green checkmark:

#What to do
1Click the "Print website URL" step in Actions
2Copy the URL (looks like the one below)
3Paste it in your browser and hit Enter
🌐 https://d1abc23xyz456.cloudfront.net
🎉Your website is live on the internet, secured with HTTPS, served globally via AWS CloudFront CDN!

Want to make changes? Just edit index.html, then run:

git add index.html
git commit -m "Update site"
git push origin main

The pipeline auto-deploys in ~2 minutes. That's CI/CD! 🚀
